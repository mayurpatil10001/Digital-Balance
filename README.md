# Digital Balance — Mental Wellness Predictor

> **Educational and research demonstration — not a medical or diagnostic tool.**

A complete, end-to-end machine learning web application that predicts a **Mental Wellness Score (0–100)** from screen-time habits, sleep quality, stress levels, exercise, and social patterns. Enter your daily routine into a polished single-page app and receive an instant score, category label, top driving factors, and personalised improvement tips — all powered by a trained scikit-learn pipeline served via FastAPI.

## Train the Model

```powershell
python model/train_model.py
```

What this does:

1. Loads and cleans `data/ScreenTime_vs_MentalWellness.csv`
   - Drops `user_id` and the trailing empty column
   - Caps physically impossible values (e.g. `screen_time_hours > 24` → 24)
   - Flags (but retains) rows where `|total − work − leisure| > 1 hr`
2. Runs EDA and saves **4 charts** to `model/plots/`
3. Trains and compares **4 candidate models** on an 80/20 split:
   - Linear Regression, Ridge Regression, Random Forest, Gradient Boosting
4. Selects the best model by test-set R² (ties broken by lowest RMSE)
5. Tunes the winner with **`RandomizedSearchCV`** (n_iter=20, 5-fold CV)
6. Saves `wellness_model.pkl`, `metrics.json`, and `feature_importance.json`
7. Prints a final summary with R², MAE, RMSE, and top 5 features

> **Expected runtime:** 30–90 seconds depending on hardware.

## Serve the Frontend

Open a **new terminal**, then:

```powershell
cd frontend
python -m http.server 8080
```

Open your browser at **http://localhost:8080**

The app will:
1. Call `GET /metadata` to populate dropdowns and the hero model chip
2. Bind all sliders with live fill-track animations
3. On form submit — `POST /predict` → animate the SVG gauge and render cards

> **Note:** `app.js` is hardcoded to call `http://127.0.0.1:8000`.  
> If your backend runs on a different port, edit the `API_BASE` constant at the top of `app.js`.

## API Reference

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/health` | Liveness check → `{"status": "ok"}` |
| `GET` | `/metadata` | Dropdown options, slider ranges, category thresholds, model info |
| `POST` | `/predict` | Accepts `WellnessInput` JSON → returns `WellnessOutput` |
| `GET` | `/docs` | Interactive Swagger UI (auto-generated by FastAPI) |

### POST /predict — Request body (`WellnessInput`)

```json
{
  "age": 28,
  "gender": "Female",
  "occupation": "Employed",
  "work_mode": "Remote",
  "screen_time_hours": 9.5,
  "work_screen_hours": 6.0,
  "leisure_screen_hours": 3.5,
  "sleep_hours": 7.0,
  "sleep_quality_1_5": 3,
  "stress_level_0_10": 6.5,
  "productivity_0_100": 65.0,
  "exercise_minutes_per_week": 120,
  "social_hours_per_week": 8.0
}
```

### POST /predict — Response (`WellnessOutput`)

```json
{
  "predicted_score": 18.4,
  "category": "Fair",
  "top_factors": ["Stress level", "Sleep quality", "Hours of sleep"],
  "tips": [
    "Try to reduce stress through mindfulness or short breaks.",
    "Aim for at least 7–8 hours of quality sleep each night."
  ],
  "disclaimer": "This is an estimated wellness indicator based on a statistical model trained on survey data. It is not a diagnosis and not a substitute for professional mental health advice."
}
```

All fields are validated by Pydantic v2. Out-of-range or missing fields return **HTTP 422**.

## Model Performance

Four models were compared on an identical 80/20 train/test split:

| Model | Test R² | Test MAE | Test RMSE | CV R² (mean ± std) |
|---|---|---|---|---|
| Linear Regression | 0.9335 | 4.009 | 5.129 | 0.931 ± 0.012 |
| **Ridge Regression** ✅ | **0.9335** | **3.995** | **5.127** | **0.932 ± 0.012** |
| Random Forest | 0.889 | 5.268 | 6.625 | 0.897 ± 0.004 |
| Gradient Boosting | 0.905 | 4.893 | 6.143 | 0.908 ± 0.011 |

**Winner: Ridge Regression** (tied highest R², lowest RMSE breaks tie).

After `RandomizedSearchCV` tuning (n_iter=20, 5-fold CV):

| Metric | Value |
|---|---|
| **Tuned Test R²** | **0.9327** |
| **Tuned Test MAE** | **4.021** |
| **Tuned Test RMSE** | **5.158** |
| **Tuned Train R²** | 0.9393 |

## Category Thresholds

Categories are derived from **training-set quartiles** of the target variable — not hardcoded:

| Category | Score Range | Training Quartile |
|---|---|---|
| **Poor** | < 3.225 | Below Q25 |
| **Fair** | 3.225 – 13.4 | Q25 – Q50 |
| **Good** | 13.4 – 30.525 | Q50 – Q75 |
| **Excellent** | ≥ 30.525 | Above Q75 |

Exact values are stored in `model/metrics.json → category_thresholds` and served dynamically via `GET /metadata`.

## Design Decisions & Assumptions

| Decision | Rationale |
|---|---|
| **Regression, not classification** | `mental_wellness_index_0_100` is continuous; categories are derived post-prediction from training quartiles |
| **Category thresholds from quartiles** | Avoids hardcoding; reflects actual data distribution (right-skewed, median ≈ 14.8) |
| **Screen-time discrepancies flagged but kept** | Rows where `|total − work − leisure| > 1 hr` are logged; data is sparse (400 rows) so no rows dropped unless physically impossible |
| **Cap rather than drop out-of-range values** | Preserves sample size; logs how many rows were capped |
| **Model loaded once at import** | Avoids per-request disk I/O; startup failure is logged with a clear message |
| **CORS set to `*`** | Local development only; restrict `allow_origins` in production |
| **Tips engine is rule-based, not ML** | Keeps tips interpretable and controllable; avoids hallucination risk |
| **`productivity_0_100` as feature, not target** | It is a user-reported behavioural metric, not the outcome variable |
| **`StandardScaler` fit on train set only** | Prevents data leakage from test set into preprocessing |
| **One-hot encoding with `drop='first'`** | Avoids dummy-variable trap for linear models |

## Known Limitations

1. **Small sample size (n = 400):** Predictions carry significant uncertainty, especially for edge-case profiles.
2. **Self-reported data:** All features and the target are self-reported; recall bias and anchoring effects are likely.
3. **Correlation ≠ causation:** Associations between screen time / sleep / stress and wellness scores do not establish causal relationships.
4. **Not clinically validated:** The `mental_wellness_index_0_100` target has not been validated against clinical measures (DSM criteria, PHQ-9, GAD-7, etc.).
5. **Right-skewed target:** Most participants score low (mean ≈ 20, median ≈ 14.8). Predictions for high-wellness users (score > 60) extrapolate from sparse data.
6. **Cross-sectional dataset:** No temporal or longitudinal inference is possible.
7. **Unknown demographic distribution:** Predictions may not generalise to all age groups, cultures, or occupational contexts.
